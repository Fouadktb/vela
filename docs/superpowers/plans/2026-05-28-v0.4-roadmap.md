# Vela v0.4 Roadmap Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and release Vela `0.4.0+6` with provider reliability, player quality, live guide, library UX, diagnostics, backup/reset, and release packaging.

**Architecture:** Keep Flutter, Drift, Riverpod, and media_kit. Add focused service/model files where needed; avoid broad rewrites of existing large screens except where small extraction reduces risk. Persist operational state in existing Drift tables where possible and use `AppSettings` for small preference/diagnostic values.

**Tech Stack:** Flutter desktop, Riverpod, Drift/SQLite, media_kit, file_selector/path_provider, GitHub Releases/manual packaging.

## Task 1: Provider Health and Catalog Stats

**Files:**

- Modify: `lib/src/catalog/catalog_repository.dart`
- Modify: `lib/src/providers/provider_repository.dart`

- [ ] **Step 1: Add provider health model**

Add immutable models:

```dart
class ProviderCatalogStats {
  const ProviderCatalogStats({
    required this.liveCount,
    required this.movieCount,
    required this.seriesCount,
    required this.episodeCount,
    required this.epgProgramCount,
  });

  final int liveCount;
  final int movieCount;
  final int seriesCount;
  final int episodeCount;
  final int epgProgramCount;
}

class ProviderHealth {
  const ProviderHealth({
    required this.provider,
    required this.stats,
    this.latestRun,
    this.nextRefreshAtMs,
  });

  final CatalogProvider provider;
  final ProviderCatalogStats stats;
  final ProviderRefreshRun? latestRun;
  final int? nextRefreshAtMs;

  bool get hasImportedCatalog =>
      stats.liveCount + stats.movieCount + stats.seriesCount > 0;
  bool get latestRefreshFailed =>
      latestRun?.status == ProviderRefreshStatus.failed;
}
```

- [ ] **Step 2: Query catalog stats**

Add `CatalogRepository.providerCatalogStats(String providerId)` that counts non-stale live/movie/series catalog items, non-stale episodes, and EPG rows. Use Drift `countAll()` queries or `customSelect` with `COUNT(*)`. Return `ProviderCatalogStats`.

- [ ] **Step 3: Query latest refresh run**

Add `ProviderRepository.latestRefreshRun(String providerId)` and `ProviderRepository.watchProviderHealth()` by composing providers, stats, latest run, and next refresh timestamp. Keep credentials redacted.

- [ ] **Step 4: Render provider health in Settings**

Replace the current flat provider list section with compact health cards showing provider name, enabled status, source type, last refresh, next refresh, counts, and last error if any. Keep existing refresh/delete/update controls.
