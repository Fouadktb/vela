# Vela v0.4 Roadmap Spec

## Goal

Vela v0.4 turns the current tester-ready IPTV player into a more complete daily-use desktop app: faster provider handling, better health visibility, stronger playback UX, real guide browsing, richer library navigation, diagnostics, backup/reset, and polished release metadata.

## Release Boundary

This release keeps the existing Flutter + media_kit architecture. It does not add paid signing, automatic in-app updates, cloud sync, account sharing, DRM playback, or provider-specific illegal-content bypasses. Updates remain manual through GitHub Releases.

## Provider Reliability

Provider setup must feel responsive even for very large Xtream catalogs. Xtream imports should save the provider only after the first catalog import succeeds. Import progress must show clear stages: validating account, loading live categories/channels, loading movie categories/items, loading series categories/items, loading available EPG, and indexing. Episode detail loading must be lazy and bounded: initial import should not block on thousands of per-series detail requests.

Settings must show provider health: enabled status, source type, auto-refresh interval, last refresh, next refresh, latest refresh result, catalog counts, EPG count, and whether credentials/API validation worked. Failed refreshes must preserve the last successful catalog.

## Playback Quality

Vela remains a single in-app player. media_kit remains the playback engine. Player controls must expose usable audio, subtitle, video track, and speed menus; keyboard shortcuts must cover common desktop behavior; buffer state must be understandable; and stalled playback must offer retry. The app must remember selected audio/subtitle/video track labels per playable item where the track exists on future playback.

The player must show next episode continuity clearly. Completed episodes should continue to the next episode, while unfinished episodes resume from the saved position.

## Live TV and EPG

Live TV needs a guide view that is useful without leaving the catalog. The guide shows channels vertically, programs horizontally, current time, currently airing program, and next program. If EPG is missing, the UI must say so and offer provider EPG refresh where possible. The existing list/detail flow remains available.

## Library UX

Movies and series need stronger detail pages. Series detail pages should select and scroll to the current/next episode. Episode progress must show watched, resume, or next-up state. Global search must search across live, movies, series, favorites, and recents without carrying category/search state between sections unexpectedly. Categories should continue to support favorite/reorder, but the UI should make favorite categories easier to reach.

## Settings, Diagnostics, Backup

Settings must become the operational hub. It needs provider health, refresh controls, data tools, update status, and diagnostics. Diagnostics export should write a local text file with app version, platform, provider summaries, latest refresh runs, playback summary, and recent non-sensitive errors. Credentials and playlist URLs must be redacted. Backup should export non-secret user data and app preferences; reset should allow clearing cache/history/catalog with explicit confirmation.

## Release Requirements

v0.4 must update app version/build metadata, keep release notes clean, and build macOS DMG plus Windows installer. Verification uses formatting, static analysis, macOS build, packaging, and the manual Windows installer workflow. No test suite is added because the project owner explicitly requested no tests.

## Acceptance Criteria

- Provider import has stage progress and does not save failed imports as usable providers.
- Settings shows provider health, catalog counts, last/next refresh, and refresh result.
- User can manually refresh provider and EPG from Settings and relevant live guide states.
- Playback controls include working menus for audio, subtitles, video tracks, speed, retry, and keyboard shortcuts.
- Track selections are remembered per playable item when matching tracks are available later.
- Live TV has a guide view with current/next program rendering from existing EPG data.
- Series and movie detail pages show richer metadata and correct progress/continue state.
- Global search can surface results across all major content types.
- Diagnostics export produces a redacted file.
- Backup/reset tools exist in Settings with confirmation.
- Version is bumped to `0.4.0+6`.
- macOS release build succeeds.
